# 项目经历

## OA 无纸化办公审批系统

**项目描述：**

本项目是一个面向企业内部办公场景的 OA 无纸化审批系统，支持可视化流程配置、动态表单设计、在线申请、审批流转、任务待办、流程追踪和实时消息通知。系统以后端自研轻量审批流引擎为核心，实现流程实例推进、任务分发、条件网关判断和流程历史记录，适用于请假、报销、采购、出差、用章等常见办公审批场景。

**技术栈：**

Vue 3、TypeScript、Vite、Pinia、Vue Router、Ant Design Vue、AntV X6、Axios、Koa、Prisma、SQLite、JWT、WebSocket

**项目职责 / 技术亮点：**

- 基于 Vue 3 + TypeScript 搭建前端工程，使用 Vue Router 实现页面路由和登录拦截，使用 Pinia 拆分认证、流程、实例、任务、通知和主题等业务状态，提升模块边界清晰度和可维护性。
- 基于 Ant Design Vue 构建 OA 工作台、流程管理、在线申请、我的申请、待办审批、任务详情和实例详情等页面，覆盖从流程配置到申请审批的完整业务链路。
- 基于 AntV X6 实现可视化审批流程设计器，支持开始节点、结束节点、用户任务、排他网关、并行网关等节点配置，并支持节点拖拽、连线编辑、属性配置和流程 JSON 持久化。
- 设计并实现动态表单配置能力，支持文本、多行文本、数字、下拉选择、日期等基础字段类型，使不同 OA 申请场景可以复用同一套申请和审批流程。
- 基于 Koa + Prisma 实现后端分层架构，按照 route、controller、service、Prisma model 拆分认证、流程定义、流程实例、审批任务和通知等模块，提升接口扩展性。
- 设计自研轻量审批流引擎，支持流程实例创建、节点流转、用户任务生成、条件表达式判断、排他网关分支选择、流程历史记录和实例完成状态更新。
- 封装 Axios 请求层，实现 JWT Token 自动注入、统一响应解析、错误提示、请求重试和登录过期处理，降低业务页面重复代码。
- 集成 WebSocket 通知能力，在用户登录后建立实时连接，为待办提醒和系统通知提供基础能力。
- 设计主题切换方案，基于 Ant Design Vue ConfigProvider、Pinia 和 localStorage 实现主色配置、明暗主题切换和刷新后主题恢复。
- 封装虚拟列表 Hook 和组件，用于审批记录、通知列表等潜在大数据量列表场景，减少非可视区域 DOM 渲染，提高页面滚动性能。

**项目边界：**

- 项目定位为轻量 OA 审批系统，不是完整 BPMN 2.0 工作流平台。
- 当前重点支持企业内部申请审批场景，不覆盖考勤、公文、邮箱、知识库等完整 OA 套件能力。
- 当前本地开发使用 SQLite，适合学习、演示和二次开发，生产环境可迁移到 MySQL 或 PostgreSQL。

**可扩展方向：**

- 审批意见、我的已办、抄送我的、申请撤回。
- 请假、报销、采购、出差、用章等 OA 申请模板。
- 流程分类、流程归档、流程版本管理。
- 用户、角色、部门和接口权限管理。
- 附件上传、通知落库和已读/未读状态。
- 流程引擎单元测试、部署文档和 Docker Compose。
