# OA 无纸化办公审批系统项目边界

## 项目定位

本项目定位为一个面向中小型企业内部办公场景的 OA 无纸化审批系统，核心目标是将常见线下申请、审批、流转、通知和归档过程线上化。

项目以“审批流”为核心能力，通过可视化流程配置、动态表单配置、在线申请、任务待办、审批处理、流程追踪和消息通知，支撑请假、报销、采购、出差、用章等典型办公审批场景。

本项目不是完整 BPMN 工作流平台，也不是企业级流程中台；当前更适合作为轻量 OA 审批系统、毕业设计、作品集项目或二次开发基础。

## 当前支持范围

### 账号与认证

- 用户注册、登录、退出。
- JWT 登录态维护。
- 前端路由登录校验。
- 后端受保护接口认证中间件。

### OA 工作台

- 数据概览入口。
- 待办任务入口。
- 我的申请入口。
- 流程设计与申请入口。
- 通知中心入口。

### 审批流程管理

- 流程定义列表。
- 新建流程定义。
- 编辑流程定义。
- 发布流程定义。
- 删除流程定义。
- 保存自定义 JSON 流程结构。

### 可视化流程设计

- 基于 AntV X6 的流程图设计。
- 支持开始节点、结束节点、用户任务、排他网关、并行网关等基础节点。
- 支持节点拖拽、连线编辑和节点属性配置。
- 支持流程图结构保存到后端。

### 动态表单配置

- 为流程配置申请表单字段。
- 支持文本、多行文本、数字、下拉选择、日期等基础字段类型。
- 支持必填、默认值、选项、说明等基础配置。

### 在线申请

- 用户选择已发布流程发起申请。
- 根据流程表单生成申请页面。
- 提交表单变量并创建流程实例。
- 支持查看我的申请记录。

### 审批任务

- 生成待办任务。
- 查询任务列表。
- 查询我的待办。
- 审批通过。
- 驳回任务。
- 转交任务。
- 查看任务详情。

### 流程实例追踪

- 查询流程实例列表。
- 查询流程实例详情。
- 查看流程变量。
- 查看流程历史记录。
- 取消运行中的流程实例。

### 消息通知

- 前端通知中心。
- WebSocket 连接能力。
- 支持后端发送测试通知。

### 主题与体验

- 明暗主题切换。
- 主色配置。
- Ant Design Vue 主题集成。
- 虚拟列表组件用于大列表场景优化。

## 明确不支持范围

### 流程引擎边界

- 不支持完整 BPMN 2.0 标准。
- 不支持 BPMN XML 导入导出。
- 不支持复杂子流程。
- 不支持定时器事件。
- 不支持边界事件、补偿事件、消息事件等完整 BPMN 事件体系。
- 不支持复杂服务编排。
- 不支持分布式事务补偿。
- 不承诺企业级高并发流程引擎能力。

### 表单平台边界

- 不支持完整低代码表单平台。
- 不支持复杂子表单。
- 不支持复杂字段联动数据源。
- 不支持自定义脚本插件体系。
- 不支持复杂报表设计器。

### OA 套件边界

- 不支持完整考勤系统。
- 不支持完整公文流转系统。
- 不支持在线文档协作。
- 不支持企业邮箱。
- 不支持即时通讯。
- 不支持知识库。
- 不支持资产管理、合同管理等完整 OA 子系统。

### 平台化边界

- 不支持多租户 SaaS。
- 不支持插件市场。
- 不支持复杂组织架构同步。
- 不支持大规模分布式部署。
- 不支持完善的审计合规体系。

## 技术边界

### 前端

- Vue 3 + TypeScript + Vite。
- Pinia 负责状态管理。
- Vue Router 负责页面路由和登录校验。
- Ant Design Vue 负责 UI 组件。
- AntV X6 负责可视化流程设计器。
- Axios 负责接口请求封装。

### 后端

- Koa + TypeScript。
- Prisma 负责数据库访问。
- SQLite 作为本地开发数据库。
- JWT 负责用户认证。
- WebSocket 负责实时通知能力。

### 数据存储

- 当前主要存储用户、流程定义、流程实例、流程历史、任务数据。
- 流程定义采用项目自定义 JSON 格式。
- 表单字段作为流程定义的一部分存储。
- 当前本地数据库使用 SQLite，更适合开发、演示和学习，不建议直接作为生产数据库。

## 推荐项目描述

本项目是一个面向企业内部办公场景的 OA 无纸化审批系统，支持可视化流程配置、动态表单设计、在线申请、审批流转、任务待办、流程追踪和实时消息通知。系统后端基于自研轻量审批流引擎实现流程实例推进、任务分发、条件网关判断和流程历史记录，前端基于 Vue 3、Ant Design Vue 和 AntV X6 实现流程设计器与审批操作界面。
